## 🔑 系統的5個核心模塊

### 1. 報頭解析 (已開始)
```
輸入：原始以太網數據包
輸出：識別是否ROCE v2，提取協議信息
關鍵：檢查EtherType=0x0800, 協議=17, 端口=4791
延遲：2個時鐘周期
```

### 2. ECN標記 (已有初版)
```
輸入：識別後的ROCE數據包
輸出：監控隊列深度，標記ECN位
關鍵：隊列深度>50時設置ECN=0b11
延遲：1個時鐘周期
```

### 3. 優先級隊列 (待開發)
```
輸入：帶有優先級的數據包
輸出：按優先級排隊和調度
關鍵：4級優先級FIFO，優先級0最高
延遲：1個時鐘周期
```

### 4. 流控制 (待開發)
```
輸入：隊列深度并下游就緒狀態
輸出：背壓信號和PAUSE幀請求
關鍵：防止隊列溢出，實現零丟包
延遲：0個時鐘周期（組合邏輯）
```

### 5. 效能監控 (待開發)
```
輸入：所有通過系統的數據包
輸出：延遲、吞吐量、丟包率統計
關鍵：每1000周期計算一次性能指標
延遲：統計性，不影響數據路徑
```

## 🎯 預期的性能指標

```
延遲 (Latency)
├─ 平均：30-50ns (3-5個周期)
├─ 最大：80-150ns
└─ 目標：< 100ns ✓

吞吐量 (Throughput)
├─ 實現：850-900 Mbps
├─ 理論最大：1024 Mbps (100MHz × 128bit)
└─ 目標：> 800Mbps ✓

丟包率 (Loss Rate)
├─ 有背壓流控：0% ✓
├─ 無背壓：可能丟包
└─ 目標：0% ✓

隊列深度
├─ 峰值：50-80數據包
├─ 平均：20-30數據包  
└─ 目標：< 100 ✓
```

## 💻 Verilog一頁速查

### AXI-Stream握手（最重要！）
```verilog
// ✅ 正確：只在握手時轉移數據
always @(posedge clk) begin
    if (s_axis_tvalid && s_axis_tready) begin
        output_data <= s_axis_tdata;
    end
end

// ✅ 背壓：正確傳播
assign s_axis_tready = m_axis_tready && (queue_not_full);
```

### FIFO隊列（最常用）
```verilog
// 空/滿判斷
wire fifo_empty = (count == 0);
wire fifo_full = (count == MAX_DEPTH);

// 更新計數
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) count <= 0;
    else if (wr && !rd) count <= count + 1;
    else if (!wr && rd) count <= count - 1;
end
```

### 復位序列（必須做對）
```verilog
always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        state <= IDLE;
        counter <= 0;
        data <= 0;    //← 復位所有暫存器！
    end else begin
        // 正常邏輯
    end
end

// 仿真中
initial begin
    rst_n = 0;
    #10;           // 保持最少1-2個周期
    rst_n = 1;
end
```

## 🐛 常見Verilog錯誤及修正

| 錯誤 | 修正 |
|------|------|
| 忘記握手 | 總是檢查 `tvalid && tready` |
| 復位不全 | 列出並復位所有暫存器 |
| 背壓丟失 | 確保 `tready` 能立即反應 |
| 未初始化 | 所有計數器初始化為0 |
| 時序錯誤 | 檢查組合邏輯環路 |
