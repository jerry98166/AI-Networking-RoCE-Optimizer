### **【核心專題：硬體加速 AI 網路壅塞控制】**

**Page 13: 專題封面**
「接下來，我想向各位主管重點分享我近期的硬體實作專題：『硬體加速的 AI 網路壅塞控制』。這是一個基於 FPGA 實作 RoCE v2 與 ECN 機制的專案，目標是將延遲控制在 100 奈秒以內。」

**Page 14: AI 訓練的瓶頸**
「我們都知道，現代的分散式 AI 訓練對於網路環境的要求極為嚴苛，需要極高的頻寬以及微秒級的同步能力，對於『掉包 (Drop)』更是零容忍。然而，傳統 TCP/IP 協定在使用 WRED 等機制時，往往是在緩衝區溢位時才被動掉包。這會導致嚴重的延遲突波和重傳代價，完全無法滿足 AI 訓練的需求。」

**Page 15: 典範轉移**
「因此，這個專案的核心在於『典範轉移』。我們將處理媒介從 CPU 的軟體協議棧，卸載到 SmartNIC 或 FPGA 等硬體上。捨棄被動的掉包，改為實作 RoCE v2 搭配主動的 ECN (明確壅塞通知)。在封包造成擁塞前，就透過硬體主動偵測佇列深度並標記，將延遲從微秒級大幅降至 100ns 以下的穩定區間。」

**Page 16: Top-Level RTL 架構**
「這是本次主題設計的 Top-Level RTL 架構，採用 128-bit 的 AXI-Stream 進行資料流傳輸。整個 Data Path 分為五個關鍵的 Pipeline 階段，包含解析、標記、優先權排程、流量控制以及效能監控。接下來我會逐一說明。」

**Page 17: Stage 1 - 封包解析**
「第一階段是零延遲的 RoCE Header Parser。當資料流入時，硬體會在第一個 Cycle 內，同步檢查 Ethertype、Protocol 以及 Destination Port。只要條件吻合，系統就能在『零延遲』的狀態下，立刻判定這是一個 RoCE 封包，不需要等待 Payload 緩衝。」

**Page 18: Stage 2 - ECN 標記**
「第二階段是壅塞控制的核心：ECN Marker。硬體邏輯會即時監控內部佇列深度。當深度超過設定的閾值（例如 50）時，硬體會觸發控制訊號，透過 Multiplexer 直接在行進中的封包修改 IPv4 的 ToS Bits，寫入 2'b11，完成 In-Flight 的標記。」

**Page 19: Stage 3 - 優先權排程**
「第三階段，為了確保帶有 ECN 標記的緊急控制封包不被一般資料塞住，因此設計了一個 Strict Priority Arbiter。系統會絕對優先處理 Priority 0 的佇列，確保關鍵的壅塞通知能夠達到『零隊頭阻塞 (Zero Head-of-line Blocking)』。」

**Page 20: Stage 4 - 流量控制與反壓**
「第四階段是 AXI-Stream 的反壓機制 (Backpressure)。為了保證絕對的零掉包，當下游系統回傳 tready = 0 (代表忙碌) 時，模組會立刻把資料 Hold 在匯流排上。同時，若內部佇列持續攀升，硬體還會觸發 Failsafe 機制，發送 Ethernet PAUSE frame 暫停外部流量。」

**Page 21: Stage 5 - 硬體遙測**
「第五階段， 100% 硬體加速的效能監控模組。它可以即時計算絕對延遲與吞吐量，並追蹤掉包數。因為這些邏輯是 completely out-of-band，所以消耗的 CPU Cycle 數為 0。」

**Page 22-24: 驗證方法論 (RTL、機制、波形)**
「完成設計後，嚴謹的驗證是不可或缺的。
（翻到 P22）首先在 RTL 合成分析上，確保內部狀態機能夠嚴格遵循同步時脈，消除了組合邏輯迴路的風險。
（翻到 P23）在機制上，嚴格遵守 AXI-Stream 的交握協定，確保在 Zone B 的壅塞狀態下，資料能穩定 Hold 住不遺失。
（翻到 P24）最後，也撰寫了 Testbench，透過 Vivado Xsim 的波形圖，成功驗證了 Reset 狀態、AXI 交握成功，以及壅塞觸發時的精準行為。」

**Page 25-27: Roadmap 與目前進度**
「這是我針對專案規劃的 Roadmap。目前 Phase 1 的核心邏輯與波形驗證已經完成。接下來的 Phase 2，我正在進行多佇列 QoS 的整合，以及高負載的壓力測試（Stress testing）。」

**Page 28: 量產級藍圖 (The Silicon Roadmap)**
「若放眼未來的量產級藍圖，除了目前的 100MHz 與 128-bit 匯流排，Phase 3 的目標將是拓寬時脈與匯流排至 10Gbps+ 的生產線速率，甚至導入由 AI 驅動的動態 ECN 閾值調整。」

**Page 29: 工程師的綜合淬鍊**
「透過這個專案，我不僅達成了從網路問題定義到ip層級驗證的閉環，並學習到了三大重點：網路協定的 Bit-level 解析能力、Cycle-accurate 的 Verilog 架構設計，以及撰寫 Testbench 分析波形的驗證方法論。」
